    **Step 2: Create `src/renderer.js` with the full rendering pipeline.**

    The module exports three functions:
    - `initRenderer()` — async, called once at startup. Initializes Shiki highlighter (via `await import('shiki')` since Shiki is ESM-only) and Mermaid (via `await import('mermaid')` + svgdom DOM polyfill). Sets `mermaidAvailable` flag to false if init fails (graceful degradation). Creates the markdown-it instance with plugins.
    - `renderMarkdown(source)` — async, called per request. Implements the two-pass strategy from RESEARCH.md: (1) regex-extract all ````mermaid` blocks, render each to SVG via mermaid.render() with unique IDs, cache results; (2) override markdown-it's fence renderer to inject cached SVGs for mermaid blocks (or fallback HTML on error), then call `md.render(source)`. Returns the rendered HTML body string wrapped in `<div class="markdown-body">`.
    - `buildPage({ filePath, bodyHtml })` — sync, wraps bodyHtml in a full `<!DOCTYPE html>` page with `<meta charset>`, `<meta viewport>`, `<title>` (escaped filePath), `<link rel="stylesheet" href="/styles/markdown.css">`, a `<header class="breadcrumb"><code>{filePath}</code></header>`, and `<main class="markdown-body">{bodyHtml}</main>`.

    Key implementation details per CONTEXT.md and RESEARCH.md:
    - `html: false` in markdown-it config (security — no raw HTML passthrough)
    - `linkify: true` in markdown-it config
    - Shiki theme: `github-dark` (complements dark background)
    - Shiki `highlight` callback uses `highlighter.codeToHtml(code, { lang, theme: 'github-dark' })`. For unknown languages, fall back to `'text'`.
    - Check `highlighter.getLoadedLanguages().includes(lang)` before highlighting; if not loaded, use `'text'`.
    - Load common languages at init: javascript, typescript, python, bash, shell, json, yaml, toml, markdown, html, css, rust, go, java, sql, diff, xml, c, cpp, ruby, php, swift, kotlin, dockerfile, graphql, plaintext.
    - Mermaid init: `createHTMLWindow()` from svgdom, assign to `globalThis.window` and `globalThis.document`, then `mermaid.initialize({ startOnLoad: false, htmlLabels: false, flowchart: { htmlLabels: false }, securityLevel: 'strict', theme: 'dark' })`.
    - Mermaid failure fallback: wrap the raw source in a Shiki-highlighted code block (lang: 'text') plus a `<p class="mermaid-error">` with the error message.
    - Two-pass render: Use regex `/^```mermaid\n([\s\S]*?)^```/gm` to find mermaid blocks BEFORE calling md.render(). Render each to SVG async. Store in a Map keyed by source content. Override `md.renderer.rules.fence` to check if the fence is `mermaid` lang and inject the cached SVG or fallback.
    - `buildPage` must escape HTML in filePath (replace `<`, `>`, `&`, `"` with entities).
    - Wrap `initRenderer` in try/catch per subsystem — if Shiki fails, highlighter stays null and code blocks get default escaping. If Mermaid fails, `mermaidAvailable` stays false and mermaid blocks render as code. Log warnings but don't crash.
