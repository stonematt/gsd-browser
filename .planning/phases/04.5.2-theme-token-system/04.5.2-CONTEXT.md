# Phase 4.5.2: Theme Token System - Context

**Gathered:** 2026-03-27
**Status:** Ready for planning

<domain>
## Phase Boundary

Extract all hardcoded hex colors in index.html, sources.html, and renderer.js into CSS custom properties. Dark theme (Catppuccin Mocha) is the default. Light theme (Catppuccin Latte) activates via `prefers-color-scheme: light`. No manual toggle — auto-detect only. Status colors (complete/active/pending) work across both themes.

</domain>

<decisions>
## Implementation Decisions

### Theme palette
- Use Catppuccin Mocha as the dark theme (replaces current GitHub-dark palette)
- Use Catppuccin Latte as the light theme
- Map current hex values to nearest Catppuccin equivalents — visual shift is intentional, not a regression bug
- Status colors: green (complete), blue (active), gray (pending) — pick Catppuccin green/blue/overlay values that work on both Mocha and Latte backgrounds

### Token naming convention
- Semantic/role-based flat names: `--bg-page`, `--bg-surface`, `--bg-overlay`, `--border-default`, `--border-muted`, `--text-primary`, `--text-secondary`, `--text-muted`, `--text-link`, `--text-bright`
- Status tokens: `--status-complete`, `--status-active`, `--status-pending`, `--status-complete-text`, `--status-active-text`, `--status-pending-text`
- Interactive tokens: `--btn-primary-bg`, `--btn-primary-text`, `--btn-hover-bg`
- No namespacing prefix (no `--color-*`) — keep it short since all tokens are colors in this phase

### Token location and sharing
- Single external CSS file: `public/styles/theme.css` — defines `:root` (dark/Mocha default) and `@media (prefers-color-scheme: light)` (Latte override)
- All three consumers (`index.html`, `sources.html`, `renderer.js` page template) link this one file
- `markdown.css` already uses some `var(--*)` from its upstream — those upstream variables get values from `theme.css` too where applicable
- Inline `<style>` blocks in HTML files keep layout/structure rules but replace all hex colors with `var(--token-name)`

### Theme switching
- Auto-only via `prefers-color-scheme` media query — no manual toggle in UI
- Success criteria requires this; a manual toggle is scope creep (future phase if wanted)

### Claude's Discretion
- Exact Catppuccin color mapping for each current hex value
- How to handle the few colors that don't map cleanly (e.g., `#484f58` — pick nearest Catppuccin overlay/surface)
- Whether `markdown.css` upstream `var()` references need overrides or just work with the new token values
- Handling of `renderer.js` inline styles (breadcrumb links) — replace with class-based styling or keep inline with `var()`

</decisions>

<specifics>
## Specific Ideas

- Catppuccin Mocha and Latte are the reference palettes — use the official hex values from catppuccin.com
- The current GitHub-dark palette is close to Mocha but not exact — this phase intentionally shifts to true Catppuccin
- Both themes should feel cohesive with the same visual weight — Latte isn't just "invert the colors"

</specifics>

## Existing Code Insights

### Reusable Assets
- `public/styles/markdown.css`: Already uses `var(--fgColor-danger)`, `var(--bgColor-danger-muted)`, `var(--borderColor-accent-emphasis)`, `var(--borderColor-muted)` — these need values provided by the new theme.css

- Single-file SPA with inline `<style>` blocks — index.html and sources.html both have extensive inline CSS with hardcoded hex
- `renderer.js` injects a few inline styles in the page template (breadcrumb nav colors)
- `MANAGEMENT_CSP` allows `'unsafe-inline'` styles — inline `var()` references will work
- Dark theme hex values used across files: `#0d1117` (bg), `#161b22` (surface), `#21262d` (border), `#30363d` (border-strong), `#c9d1d9` (text), `#f0f6fc` (bright), `#58a6ff` (link), `#8b949e` (muted), `#484f58` (dimmed), `#238636`/`#2ea043` (green/success)

### Integration Points
- `public/index.html` — ~80+ hardcoded hex color values to replace with `var()` references
- `public/sources.html` — ~20+ hardcoded hex values
- `src/renderer.js` — 2 inline style hex values in breadcrumb template
- `public/styles/markdown.css` — already uses some `var()`, needs token values provided
- New file: `public/styles/theme.css` — single source of truth for all color tokens
