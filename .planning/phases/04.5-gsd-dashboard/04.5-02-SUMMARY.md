## Decisions Made

- Renamed `#sidebar` to `#browse-sidebar` and `#content` to `#browse-content` to properly scope them to the browse view. This avoids ID collisions in the multi-view SPA and is semantically clearer.
- Dashboard re-fetches `/api/dashboard` fresh on every view show — consistent with the existing "fresh-on-request" philosophy throughout the app (no live-reload, fresh data each access).
- Sources list is loaded lazily when the browse view is first entered, rather than at page load — avoids blocking the dashboard from appearing.
- Sidebar toggle in browse view now targets `#view-browse.sidebar-hidden` class (not `body.sidebar-hidden`) since the layout grid is on the view element, not the body.
