**Add to src/server.js inside createServer():**

1. Define `MANAGEMENT_CSP` constant:
```javascript
const MANAGEMENT_CSP = "default-src 'self'; script-src 'self'; object-src 'none'";
```

2. Add `GET /api/sources` route:
   - Call `listSources()` from sources.js (re-scans conventions fresh)
   - Return `{ sources: [...] }` with each source having name, path, available, conventions
   - Apply MANAGEMENT_CSP via per-route onSend hook

3. Add `POST /api/sources` route:
   - Read `request.body` for `{ path: string, name?: string }`
   - Call `addSource(request.body.path, { name: request.body.name })`
   - If `ok: true`, return 201 with `{ source, conventions }`
   - If `ok: false` and reason is `duplicate`, return 409 with `{ error: result.message }`
   - If `ok: false` for other reasons (invalid path, etc.), return 400 with `{ error: result.message }`
   - Apply MANAGEMENT_CSP via per-route onSend hook
   - IMPORTANT: The sources array used by routes needs to be refreshed after add/remove. Store the sources array in a mutable variable that createServer closes over, and update it after add/remove. OR, since file routes already work with the sources passed at creation time, simply let the management page tell the user to restart. Simpler approach: make the routes re-read from config on each request for the file-serving routes (consistent with fresh-from-disk philosophy). Recommended: store `let activeSources = sources` in createServer scope, update after add/remove API calls, and have /file and /render use activeSources.

4. Add `DELETE /api/sources/:identifier` route:
   - Read `request.params.identifier` (URL-decoded by Fastify automatically)
   - Call `removeSource(identifier)`
   - If `ok: true`, return 200 with `{ removed: result.removed }`
   - If `ok: false` and reason is `not-found`, return 404 with `{ error: result.message }`
   - If `ok: false` and reason is `ambiguous`, return 409 with `{ error: result.message, matches: result.matches }`
   - Apply MANAGEMENT_CSP via per-route onSend hook
   - Update activeSources after successful removal

5. Add `GET /sources` route (serves the management page):
   - Read `public/sources.html` and serve it as `text/html; charset=utf-8`
   - Apply MANAGEMENT_CSP via per-route onSend hook (allows the embedded script to run)
   - Register this route AFTER @fastify/static — explicit routes shadow static for same paths (per research)

**Create public/sources.html:**

A minimal, functional management page. Dark theme to match the markdown rendering aesthetic. No framework — vanilla HTML + JS.

Structure:
- Title: "gsd-browser - Sources"
- A table/list showing all registered sources (name, path, status, conventions)
- Each source row has a "Remove" button (no confirmation dialog — per locked decision)
- A form at top or bottom: text input for path + optional name input + "Add" button
- Status messages (success/error) displayed inline

JavaScript (embedded in `<script>` tag):
- On load: `fetch('/api/sources')` and populate the source list
- Add button: `POST /api/sources` with `{ path, name }`, refresh list on success, show error on failure
- Remove button: `DELETE /api/sources/{name}` (encodeURIComponent the name), refresh list on success

Styling (embedded in `<style>` tag):
- Dark background (#0d1117 or similar to match github-markdown-dark)
- Monospace font for paths
- Simple table layout
- Minimal — this is a bridge UI, Phase 4 replaces it
