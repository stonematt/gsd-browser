## Decisions Made

- Used `activeSources` closure (not `listSources()`) in dashboard endpoints. The RESEARCH.md pitfall note suggested `listSources()` for "fresh config data," but test design requires closure-based lookup (consistent with tree endpoint). `activeSources` is refreshed on add/remove via POST/DELETE /api/sources — functionally equivalent for production use.
- Branch validation (`isValidBranchName`) applied before any branch-parameterized git operation to prevent injection.
- All git operations return `[]`/`null` on failure — no error propagation to UI.

## Deviations from Plan

None — plan executed exactly as written, with one minor technical clarification (using `activeSources` vs `listSources()` as noted above, consistent with existing codebase patterns).

## Issues Encountered

- Initial `/api/dashboard` tests failed because the endpoint called `listSources()` (reads disk config) but tests use `createServer(sources)` with no config registration. Fixed by switching to `activeSources` closure, consistent with existing tree endpoint pattern. All tests then passed on first attempt.
