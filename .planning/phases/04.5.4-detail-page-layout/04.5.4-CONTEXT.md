# Phase 4.5.4: Detail Page Layout - Context

**Gathered:** 2026-03-28
**Status:** Ready for planning

<domain>
## Phase Boundary

The project detail page switches from a horizontal-timeline + 2-column docs panel to a three-column layout: vertical phase navigator (200px, collapsible) | file list (180px) | document content (flex). Completed phases collapse into a single summary row by default. The phase navigator column hides/shows with a CSS transition and persists collapse state in localStorage. YAML frontmatter is stripped from all rendered markdown output so the metadata card is the only representation.

</domain>

<decisions>
## Implementation Decisions

### Frontmatter stripping
- Strip YAML frontmatter (`---\n...\n---`) from ALL files in the `/render` endpoint, not just PLAN.md
- Server-side stripping before markdown-it processes the content — the HTML fragment never contains raw frontmatter
- Frontmatter data is already parsed separately via `parsePlanFrontmatter()` and returned as structured `planDetails[]` in the API — stripping the rendered version eliminates duplication
- Simple regex: `/^---\n[\s\S]*?\n---\n?/` applied to file content before rendering

### Layout and grid (from design reference)
- Three-column CSS grid: `grid-template-columns: 200px 180px minmax(0, 1fr)`
- When col 1 collapsed: `grid-template-columns: 0px 180px minmax(0, 1fr)` with ~200ms ease CSS transition
- Replaces the current horizontal timeline (`#phase-timeline-container`) and 2-column `#docs-panel`

### Phase navigator (col 1)
- Vertical list replacing horizontal timeline entirely
- Completed phases section collapses by default: single row with chevron + "N phases complete" + mini dot strip showing branching shape
- Active/future phases always visible below a horizontal divider
- Sub-phases indented by depth: 22px per level
- Dot sizes scale by plan count (0=12px, 1-2=14px, 3-4=16px, 5+=20px); sub-phase dots 4px smaller (min 8px)
- Selected phase: 2px left border accent + highlighted background
- 2px wide, 4-6px tall vertical connector bars between phase rows, indented to match depth

### Col 1 collapse behavior
- Toggle button at top-left of col 2 (not col 1): `◀`/`▶`, 22px square
- Collapse state persists in localStorage (global, not per-project)
- CSS transition on grid-template-columns

### File list (col 2) — from design reference
- Plans and summaries grouped above a divider (paired: PLAN then SUMMARY per plan number)
- Context, research, validation docs below the divider
- File status icons: `▸` (accent) for PLAN, `✓` (complete) for SUMMARY, `◇` (muted) for CONTEXT, `◎` (muted) for RESEARCH/VALIDATION
- Plan completion inferred by SUMMARY.md existence

### Document content (col 3) — from design reference
- Metadata card (structured) shown only for PLAN.md files — already built in 4.5.1
- Rendered markdown body below, with frontmatter now stripped server-side
- Same typography as browse view content pane

### Claude's Discretion
- Connector bar exact dimensions and spacing between phase rows
- Mini dot strip rendering in the collapsed completed section
- Transition timing and easing for col 1 collapse
- File list sort order within groups
- How to handle edge cases (0 completed phases, 0 active phases)

</decisions>
