    - parsePhaseDir('04.5-gsd-dashboard') returns { num: 4.5, numStr: '04.5', slug: 'gsd-dashboard', dir: '04.5-gsd-dashboard' }
    - parsePhaseDir('invalid') returns null
    - getPhaseInfo on dir with 2 PLANs, 2 SUMMARYs, 1 VERIFICATION returns { status: 'complete', planCount: 2, completedPlans: 2 }
    - getPhaseInfo on dir with 2 PLANs, 0 SUMMARYs returns { status: 'in-progress', planCount: 2, completedPlans: 0 }
    - getPhaseInfo on empty dir (no plans) returns { status: 'pending', planCount: 0, completedPlans: 0 }
    - getBranchesWithPlanning returns [] for non-git directory (no error)
    - isValidBranchName('main') returns true; isValidBranchName('feat; rm -rf /') returns false
  </behavior>
  <action>
    Add helper functions to src/server.js (above createServer):

    1. `parseStateMd(content)` — Regex-based two-level YAML frontmatter parser. Match `---\n...\n---` block, parse top-level `key: value` and indented `  subkey: value` lines. Strip quotes from values. Return object or null. Use the verified pattern from RESEARCH.md Pattern 1.

    2. `parsePhaseDir(dirName)` — Extract phase number and slug from directory names like `01-foundation` or `04.5-gsd-dashboard`. Use regex `/^(\d+(?:\.\d+)?)-(.+)$/`. Return `{ num, numStr, slug, dir }` or null. Use Pattern 3 from RESEARCH.md.

    3. `getPhaseInfo(phaseDirPath)` — Read directory, count files matching `*-PLAN.md` and `*-SUMMARY.md` patterns, check for `*-VERIFICATION.md`. Determine status: complete (has verification OR all plans have summaries), in-progress (has plans but incomplete), pending (no plans). Return `{ planCount, completedPlans, status, files }` or null. Use Pattern 2 from RESEARCH.md.

    4. Git helpers at top of file:
       ```javascript
       const { execFile } = require('node:child_process');
       const { promisify } = require('node:util');
       const execFileAsync = promisify(execFile);
       ```
       - `isValidBranchName(name)` — `/^[a-zA-Z0-9._\-/]+$/.test(name)`
       - `getBranchesWithPlanning(repoPath)` — List branches via `git -C <path> branch --format=%(refname:short)`, then check each with `git cat-file -e <branch>:.planning/STATE.md`. Return array of branch names. Return `[]` on any failure.
       - `readFileFromBranch(repoPath, branch, filePath)` — `git show <branch>:<filePath>`. Return content or null.
       - `listDirFromBranch(repoPath, branch, dirPath)` — `git ls-tree --name-only <branch>:<dirPath>`. Return array or [].

    All git operations: wrap in try/catch, return null/[] on failure. Use execFile (NOT exec) — no shell injection.

    5. `buildPhaseList(sourcePath)` — Read `.planning/phases/` directory, parse each dir name, get phase info, sort by numeric phase number. Return array of phase objects.

    6. `buildQuickLinks(sourcePath)` — Check existence of PROJECT.md, STATE.md, ROADMAP.md in `.planning/`. Return array of `{ name, path, exists }`.

    Export `parseStateMd`, `parsePhaseDir`, `getPhaseInfo`, `isValidBranchName` from module.exports for testing.

    - Create fixtures in before() hook: a `.planning/STATE.md` with YAML frontmatter, `.planning/phases/01-test/` with PLAN and SUMMARY files, `.planning/phases/02-wip/` with only PLAN files.
    - Test each helper function behavior per the behavior block above.
    - For git tests: use the actual test repo (PROJECT_ROOT) since it's a git repo. Test getBranchesWithPlanning against a non-git temp dir to verify graceful fallback.

    - GET /api/projects/:name/detail returns { source, state, phases, branch }

    - All new endpoints have MANAGEMENT_CSP via onSend hook

  <action>
    Add three new Fastify routes inside `createServer()` in src/server.js. Place them after existing `/api/sources` routes but before `/sources` and `/file` routes.

    **GET /api/dashboard:**
    - Use `listSources()` (same pattern as `/api/sources`) to get fresh source data
    - For each available source, try to read `.planning/STATE.md`
    - If STATE.md exists: parse with `parseStateMd()`, build phase list with `buildPhaseList()`, build quick-links with `buildQuickLinks()`, add to `projects` array
    - If no STATE.md: add to `other` array with just `{ name, path }`
    - Return `{ projects, other }`
    - Use `onSend` hook for MANAGEMENT_CSP (same as all other API routes)
    - Use the code pattern from RESEARCH.md "Dashboard API Endpoint" section

    **GET /api/projects/:name/detail:**
    - Find source by name from `listSources()` result
    - If not found or not available: return 404
    - Check `?branch=` query param. If present and valid branch name:
      - Read STATE.md from that branch via `readFileFromBranch()`
      - List phase dirs from that branch via `listDirFromBranch()`
      - For each phase dir, get info via `git ls-tree` to count plan/summary files
    - If no branch param (default): read from filesystem directly
    - Get branches with planning via `getBranchesWithPlanning()` (on-demand per CONTEXT.md decision)
    - Return `{ source: { name, path }, state, phases, branch, branches }`

    **GET /api/projects/:name/branches:**
    - Find source by name
    - If not found: return 404
    - Call `getBranchesWithPlanning(source.path)`
    - Return `{ branches }`
