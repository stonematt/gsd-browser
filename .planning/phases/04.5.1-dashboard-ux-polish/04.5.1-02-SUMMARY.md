## Decisions Made
- `groupPhases()` uses a separate wrapper div per group so flex-wrap wraps at group boundaries cleanly
- `makePhaseNode()` avoids code duplication between parent and child node creation
- Orphaned sub-phases (no matching parent in data) fall back to rendering as peer groups
- Pending phase merge is guarded: skipped entirely if `project.dependencies` is empty/undefined
- `parseRoadmapPhaseNames()` added to server.js rather than client-side parsing — server already reads ROADMAP.md for other purposes
- Card mini-timeline dedup fixed with `parseFloat()` comparison — zero-padded strings ("04") now match non-padded ("4") correctly
- In-progress phase distinguished with blue text + glow ring to match the active/selected visual language already established
