<task type="auto">
  <name>Task 1: Build the three-panel SPA shell</name>
  <files>public/index.html, test/server.test.js</files>
  <action>
    **Complete rewrite of public/index.html** as a single-file SPA with inline `<style>` and `<script>` (following the established pattern from the existing index.html and sources.html).

    **HTML Structure:**
    ```html
    <header id="app-header">
      <button id="sidebar-toggle">...</button>
      <nav id="breadcrumb">source / path / file.md</nav>
      <a href="/sources">Manage Sources</a>
    </header>
    <aside id="sidebar">
      <div id="source-switcher">
        <select id="source-select">...</select>
      </div>
      <div id="tree"><!-- file tree populated by JS --></div>
    </aside>
    <main id="content"><!-- rendered markdown injected here --></main>
    ```

    **CSS Layout (CSS Grid per research Pattern 1):**
    - Body: `display: grid; grid-template-rows: 44px 1fr; grid-template-columns: 260px 1fr; grid-template-areas: "header header" "sidebar content"; height: 100vh; overflow: hidden;`
    - `#app-header`: `grid-area: header; background: #161b22; border-bottom: 1px solid #21262d;`
    - `#sidebar`: `grid-area: sidebar; background: #161b22; border-right: 1px solid #21262d; overflow-y: auto;`
    - `#content`: `grid-area: content; background: #0d1117; overflow-y: auto; padding: 24px;`
    - Collapsed state: `body.sidebar-hidden { grid-template-columns: 0 1fr; }` + `body.sidebar-hidden #sidebar { display: none; }`
    - Dark theme colors: background #0d1117, text #c9d1d9, sidebar #161b22, border #21262d, link #58a6ff, muted #8b949e, bright #f0f6fc

    **CSS for file tree:**
    - `.tree-node`: padding-left based on depth (16px per level)
    - `.tree-dir`: cursor pointer, muted text (#8b949e), with expand/collapse indicator (triangle character)
    - `.tree-dir--convention`: brighter text or badge to distinguish from regular dirs
    - `.tree-dir.collapsed`: indicator rotated (-90deg)
    - `.tree-file`: color #58a6ff, no underline by default, display block
    - `.tree-file:hover`: underline
    - `.tree-file.active`: background highlight (#1f6feb20 or similar), left border accent

    **CSS for source switcher:**
    - `#source-select`: full-width select in sidebar header area, dark-styled (background #0d1117, border #21262d, color #c9d1d9)
    - Padding around the select for visual spacing

    **CSS for header:**
    - Flexbox row: sidebar toggle on left, breadcrumb in center/left, "Manage Sources" link on right
    - Breadcrumb uses monospace font, muted separators (#8b949e), filename in bright text (#f0f6fc)

    **JavaScript Application Logic (all in inline `<script>`):**

    The entire app.js should be under 300 lines per research recommendation. Key functions:

    1. **escapeHtml(str)** — copy from existing pattern (replace &, <, >, ")

    2. **parseHash(hash)** — extract sourceName and filePath from `#/source-name/path/to/file.md`
       - Strip leading `#/`
       - First segment = sourceName, rest = filePath
       - Handle edge cases: empty hash, source-only hash, full path

    3. **navigateTo(sourceName, filePath)** — update URL hash via `history.pushState(null, '', '#/' + sourceName + '/' + filePath)`, then call loadFile

    4. **State management:**
       - `let currentSource = null;` — active source name
       - `let sourcesList = [];` — cached list of available sources
       - `const lastViewedFile = new Map();` — session memory per source (per locked decision)
       - `let treeData = null;` — cached tree for active source

    5. **loadSources()** — fetch `/api/sources`, filter to available, populate `#source-select` dropdown
       - Each option: `src.name + '  (' + src.path + ')'` (per research)
       - Set `select.value` to current source if any

    6. **loadTree(sourceName)** — fetch `/api/sources/${sourceName}/tree`, call buildTreeDOM
       - Clear `#tree` container first (rebuild on every source switch per research Pitfall 4)

    7. **buildTreeNode(node, sourceName, depth)** — recursive DOM builder
       - Files: create `<a>` with class `tree-file`, `data-path`, `data-source`, href = `#/sourceName/path`
       - Directories: create `<button>` with class `tree-dir` + toggle click handler
         - Convention dirs: add `tree-dir--convention` class, start expanded
         - Non-convention dirs: start collapsed (`childContainer.hidden = true`, toggle class `collapsed`)
       - Click handler on file links: `e.preventDefault(); lastViewedFile.set(sourceName, filePath); navigateTo(sourceName, filePath);`

    8. **loadFile(sourceName, filePath)** — fetch `/render?path=${encodeURIComponent(filePath)}&fragment=true`
       - Start 200ms timer for loading indicator (per locked decision: no spinner unless >200ms)
       - On success: set `#content` innerHTML, scroll to top, update breadcrumb, call setActiveFile
       - On error: show error state in content pane
       - Update breadcrumb: `sourceName / path / segments / file.md`

    9. **setActiveFile(filePath)** — highlight file in tree + auto-expand parents (VS Code pattern)
       - Remove `.active` from all `.tree-file` elements
       - Find matching `[data-path="${CSS.escape(filePath)}"]`
       - Add `.active` class
       - Walk up DOM, unhide all `.tree-children` ancestors, remove `.collapsed` from dir toggles
       - `scrollIntoView({ block: 'nearest' })`

    10. **switchSource(sourceName)** — called from select onchange
        - Set `currentSource = sourceName`
        - loadTree, then load last-viewed file or README.md or first .md in tree
        - Handle missing README.md: check tree for README.md node, if not found, find first file node (BFS), if tree empty show empty state hint

    11. **toggleSidebar()** — `document.body.classList.toggle('sidebar-hidden')`

    12. **Boot sequence (DOMContentLoaded):**
        - Call loadSources()
        - Parse initial hash with parseHash(window.location.hash)
        - If hash has sourceName: switchSource to it (and load filePath if present)
        - If no hash: default to first source in list
        - Add popstate listener for back/forward
        - Add change listener on #source-select for source switching
        - Add click listener on #sidebar-toggle for collapse

    **Content pane styling:**
    - Link `<link rel="stylesheet" href="/styles/markdown.css">` in the `<head>` for the content pane's rendered markdown
    - The content pane has the `.markdown-body` div injected from the fragment response
    - Content pane should have `padding: 24px; max-width: 980px;` or similar readable width (reuse markdown.css max-width if applicable)

    **Add smoke tests to test/server.test.js:**
    ```
    - Test: GET / response contains id="sidebar" element
    - Test: GET / response contains id="content" element
    - Test: GET / response contains id="app-header" element
    - Test: GET / response contains id="source-select" element
    - Test: GET / response contains dark theme background color (#0d1117)
    - Test: GET / has MANAGEMENT_CSP (already exists but verify still passes)
    ```

    **Critical anti-patterns to avoid (from research):**
    - Use `textContent` for all file names, directory names, and breadcrumb segments — NEVER innerHTML with user-controlled content (Pitfall 5)
    - Do NOT change CSP for the GET / route — MANAGEMENT_CSP is already applied (Pitfall 1)
    - Hash routing only — no path-based URLs (Pitfall 3)
    - Rebuild entire #tree on source switch to avoid data-path collisions (Pitfall 4)
