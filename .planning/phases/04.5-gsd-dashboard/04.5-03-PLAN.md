  <name>Task 1: Implement project detail view with phase timeline, documentation panel, and branch selector</name>
  <files>public/index.html</files>
  <action>
    Expand the `#view-project` placeholder in `public/index.html` (created in Plan 02) into a full project detail page.

    **HTML structure for #view-project:**
    ```html
    <div id="view-project" class="view" hidden>
      <!-- Back link to dashboard -->
      <div id="project-header">
        <a href="#/" id="back-to-dashboard">&larr; Dashboard</a>
        <h2 id="project-title"></h2>
        <!-- Branch selector (hidden by default, shown when branches > 1) -->
        <div id="branch-bar" hidden>
          <select id="branch-select" aria-label="Select branch"></select>
        </div>
      </div>

      <!-- Phase timeline (horizontal bar with dots) -->
      <div id="phase-timeline" class="phase-timeline"></div>

      <!-- Documentation panel (split: file list left, rendered content right) -->
      <div id="docs-panel">
        <div id="phase-file-list"></div>
        <div id="phase-content"></div>
      </div>
    </div>
    ```

    **CSS additions (inline in existing <style> block):**
    - Phase timeline CSS from RESEARCH.md (`.phase-timeline`, `.phase-node`, `.phase-connector`, `.phase-dot`, `.phase-dot--complete/active/pending`, `.phase-label`, `.phase-progress`)
    - Active phase indicator: use a ring/glow effect on the selected phase node (e.g., `box-shadow: 0 0 0 3px rgba(88, 166, 255, 0.3)`)
    - `#project-header`: flex row with back link, title, branch selector
    - `#docs-panel`: two-column grid layout (`grid-template-columns: 220px 1fr`), full remaining height
    - `#phase-file-list`: scrollable list with `.phase-file-item` links, dark surface background (#161b22), border-right
    - `#phase-content`: scrollable content area with padding, same styling as browse view content pane
    - `.phase-file-item`: clickable items styled like tree files (#58a6ff text, hover underline, active highlight)
    - `#branch-bar`: flex row with select dropdown matching source-select styling

    **JavaScript: loadProjectDetail(sourceName, branch):**
    1. Show loading state in `#view-project`
    2. Fetch `/api/projects/${encodeURIComponent(sourceName)}/detail` (append `?branch=${branch}` if branch is specified)
    3. Parse response JSON
    4. Set project title (`#project-title`) to source name
    5. Call `renderTimeline(phases)` to build the phase timeline
    6. If `branches.length > 1`: show `#branch-bar`, populate `#branch-select` with options, set current value to response `branch`
    7. If `branches.length <= 1`: hide `#branch-bar` (per Claude discretion: only show when > 1 branch)
    8. Auto-select the first "in-progress" phase, or the last "complete" phase, or the first phase
    9. Call `selectPhase(phaseObj)` to load its docs

    **JavaScript: renderTimeline(phases):**
    - Clear `#phase-timeline`
    - For each phase, create:
      - A `.phase-node` div containing:
        - A `.phase-dot` div with status class (`.phase-dot--complete`, `.phase-dot--active` for in-progress, `.phase-dot--pending`)
        - A `.phase-label` span with "Phase {numStr}" or slug (truncated)
        - A `.phase-progress` span with "{completedPlans}/{planCount} plans"
      - A `.phase-connector` div between nodes (not after last)
    - Each node gets a click handler calling `selectPhase(phase)`
    - Store currently selected phase to highlight it

    **JavaScript: selectPhase(phaseObj):**
    1. Update active phase highlight in timeline (add/remove `.phase-node--selected` class)
    2. Clear `#phase-file-list`
    3. Filter `phaseObj.files` to only `.md` files
    4. For each file, create a clickable item in `#phase-file-list`
    5. Auto-select the first file and load it
    6. Call `loadPhaseFile(sourceName, phaseDirPath, fileName)` for the selected file

    **JavaScript: loadPhaseFile(sourceName, phaseDir, fileName):**
    1. Build the absolute file path: the source path + `.planning/phases/<phaseDir>/<fileName>`
    2. Fetch `/render?path=${encodeURIComponent(filePath)}&fragment=true`
    3. If 200: inject HTML into `#phase-content`
    4. If error: show error message in `#phase-content`
    5. Highlight active file in `#phase-file-list`

    **JavaScript: Branch switching:**
    - Wire `#branch-select` change event
    - On change: call `loadProjectDetail(sourceName, selectedBranch)`
    - Update hash to `#/project/<name>/<branch>` via `history.pushState`
    - When reading from non-default branch, the detail endpoint returns phase data from that branch's `.planning/` (server handles this via git commands)

    **Router integration:**
    - In the `showView('project')` case of the router (from Plan 02):
      - Extract `sourceName` and optional `branch` from parsed hash
      - Call `loadProjectDetail(sourceName, branch)`
    - Ensure back/forward navigation works: `popstate` handler must handle `#/project/<name>` and `#/project/<name>/<branch>`

    **Edge cases:**
    - Source with no phases: show timeline area as empty with message "No phases found"
    - Phase with no .md files: show file list as empty with message "No documentation files"
    - Branch operations fail silently (per user decision): if branches array is empty, simply don't show branch UI
    - File render fails: show error in `#phase-content`, don't crash the view
    - Use `escapeHtml()` for all dynamic text content
