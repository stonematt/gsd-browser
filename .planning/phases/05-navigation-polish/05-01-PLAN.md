  <files>package.json, src/renderer.js, test/renderer.test.js</files>
  <behavior>
    - NAV-05-1: h2 with text "Hello World" renders with id="hello-world" attribute
    - NAV-05-2: h3 and h4 headings also get id attributes
    - NAV-05-3: Two headings with same text get unique IDs (second gets "-1" suffix)
    - NAV-05-4: Each heading has a permalink anchor element with class "header-anchor"
    - NAV-06-1: renderMarkdown() on a doc with 3 h2 headings returns HTML containing a details.doc-toc element before the markdown-body div
    - NAV-06-2: TOC contains an anchor for each heading with correct href="#slug"
    - NAV-06-3: renderMarkdown() on a doc with 0-1 headings does NOT contain doc-toc
    - NAV-06-4: TOC respects heading hierarchy — h3 items have padding-left indent
    - NAV-06-5: headingsBuffer is cleared between consecutive renderMarkdown() calls (no bleed)
  </behavior>
  <action>
1. Run `npm install markdown-it-anchor` to add the dependency.

2. In `src/renderer.js`:
   - Add `const anchor = require('markdown-it-anchor');` at the top.
   - Add a module-level `let headingsBuffer = [];` alongside existing module vars.
   - In `initRenderer()`, after `md.use(footnote)`, add:
     ```
     md.use(anchor, {
       level: [2, 3, 4],
       permalink: anchor.permalink.ariaHidden({ placement: 'before' }),
       callback: (token, info) => {
         headingsBuffer.push({
           level: parseInt(token.tag.slice(1)),
           slug: info.slug,
           title: info.title
         });
       }
     });
     ```
   - Add a `buildTocHtml(headings)` function that:
     - Returns empty string if headings.length < 2
     - Builds a `<details class="doc-toc"><summary>Table of Contents</summary><nav><ul>...</ul></nav></details>` string
     - Each heading becomes an `<li>` with `<a href="#slug">title</a>`
     - h3/h4 items get `style="padding-left:16px"` (or 32px for h4) relative to the minimum heading level
     - Uses `escapeHtml()` on title text
   - In `renderMarkdown()`, AFTER the Mermaid pre-pass awaits and BEFORE `md.render()`:
     - Clear the buffer: `headingsBuffer.length = 0;`
   - After `const bodyHtml = md.render(source);`:
     - Copy headings: `const headings = [...headingsBuffer];`
     - Build TOC: `const tocHtml = buildTocHtml(headings);`
     - Return: `return tocHtml + '<div class="markdown-body">' + bodyHtml + '</div>';`
     (TOC is OUTSIDE the markdown-body div so it sits above the content)

3. In `test/renderer.test.js`, add two new describe blocks after existing tests:
   - `describe('NAV-05: Heading anchors', () => { ... })` with tests for:
     - h2 gets id attribute
     - h3 and h4 get id attributes
     - Duplicate headings get unique suffixes
     - Permalink anchor element present with header-anchor class
   - `describe('NAV-06: Inline TOC', () => { ... })` with tests for:
     - Doc with 3+ headings contains doc-toc details element
     - TOC links have correct href="#slug"
     - Doc with 0-1 headings has no doc-toc
     - Consecutive renders don't bleed headings (render doc A, render doc B, check B's TOC only has B's headings)

CRITICAL: Clear headingsBuffer AFTER all await calls (Mermaid pre-pass) and BEFORE md.render() — this prevents concurrency interleave. Do NOT await anything between the clear and md.render().
  </action>
