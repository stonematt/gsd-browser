  <name>Task 1: Restructure CLI for subcommand dispatch</name>
  <files>bin/gsd-browser.cjs</files>
  <action>
Rewrite bin/gsd-browser.cjs to support subcommand routing while maintaining backward compatibility.

**Subcommand dispatch logic:**
1. Parse args with minimist, adding `string: ['name']` and `alias: { n: 'name' }` for the `--name` flag
2. Check `args._[0]` for subcommands: `add`, `remove`, `list`
3. If subcommand is `add`:
   - `targetPath = args._[1] || '.'` (default to cwd per locked decision)
   - Call `addSource(targetPath, { name: args.name })` from sources.js
   - If `ok: true`, print: `Added {source.path}\n  Found: {conventions.join(', ')}` (or `  No conventions found` if empty). Convention names should have trailing `/` for directories (.planning/, docs/) and no trailing for files (README.md)
   - If `ok: false`, print error to stderr and exit 1
4. If subcommand is `remove`:
   - `target = args._[1]` — required, exit 1 with usage if missing
   - Call `removeSource(target)` from sources.js
   - If `ok: true`, print: `Removed {removed.name} ({removed.path})`
   - If `ok: false` and reason is `ambiguous`, print matches and ask user to specify by path
   - If `ok: false` and reason is `not-found`, print error to stderr and exit 1
5. If subcommand is `list`:
   - Call `listSources()` from sources.js
   - Format as aligned table with columns: NAME, PATH, STATUS, CONVENTIONS (docker-ps style)
   - Use `String.padEnd()` for column alignment
   - STATUS: "available" or "missing"
   - CONVENTIONS: comma-separated convention names (with trailing / for dirs) or dash if none
   - Print to stdout
6. If no subcommand (or args._[0] is not add/remove/list):
   - Load sources via `loadConfig()` + `enrichSourcesWithConventions()`
   - Filter to available sources only
   - If zero available sources: print helpful message to stderr suggesting `gsd-browser add <path>`, exit 1
   - Warn to stderr for each missing source (per locked decision)
   - Pass sources array to `start(port, sources, { open: args.open })`

**Update USAGE string** to reflect new subcommands:

General:

Remove the old "directory argument required" validation — the server now reads from config, not from a positional arg.

**Migrate src/server.js:**

1. Change `createServer(registeredRoot)` signature to `createServer(sources)` where `sources` is an array of `{ name, path, available, conventions }` objects. Keep only available sources.

2. Add a multi-source path check helper inside server.js (do NOT modify isPathAllowed in filesystem.js — per research anti-pattern):
```javascript
async function findSourceForPath(requestedPath, sources) {
  for (const source of sources) {
    const allowed = await isPathAllowed(requestedPath, source.path);
    if (allowed) return source;
  }
  return null;
}
```

3. Update GET /file route:
   - Replace `isPathAllowed(requestedPath, registeredRoot)` with `findSourceForPath(requestedPath, sources)`
   - If no source found, return 403 with `allowed: sources.map(s => s.path)`
   - Use the matched source's path as the base for `path.resolve()` and `fs.stat()`

4. Update GET /render route:
   - Same pattern — loop through sources to find which one contains the requested path
   - For the traversal check (fast sync path), check against all source roots
   - For `readFileIfAllowed()`, pass the matched source's path as the root

5. Update GET / route:
   - Try README.md from the FIRST available source (or iterate sources looking for one with a README.md)
   - Fallback to directory listing of first source

6. Change `start(port, registeredRoot, options)` to `start(port, sources, options)`:
   - Accept sources array instead of single root
   - Call `enrichSourcesWithConventions(sources)` to re-scan conventions at startup (per locked decision: fresh-from-disk philosophy)
   - Filter to `available: true` only
   - Update startup message: print each source name and path, or "Serving N sources" summary
   - Pass enriched, filtered sources to `createServer(sources)`
