## Decisions Made

- Regex changed from `(\d+(?:\.\d+)?)` (zero-or-one optional group) to `(\d+(?:\.\d+)*)` (zero-or-more) — handles unlimited depth without further changes
- `num` field (parseFloat) kept in parsePhaseDir for backward compatibility; `numStr` is canonical identity per plan spec
- `comparePhaseNums` uses `.split('.').map(Number)` with index-by-index comparison and `|| 0` for missing segments
- `phaseGoals` parsing: tracks `currentPhase` per heading; sets to null after capturing first `**Goal**` line; prevents bleed from one phase to the next
- `requirementCount` uses `Set()` dedup across all `planDetails[].requirements` — consistent with plan spec
